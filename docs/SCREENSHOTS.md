# Visual Artifacts

This repository is an architecture and infrastructure showcase. It does not claim a public product UI screenshot.

## Diagrams

- `assets/diagrams/architecture.svg`
- `assets/diagrams/architecture.mmd`

## Visual Standard

- Use architecture diagrams as the primary visual artifact.
- Keep diagrams focused on boundaries: wallet, transaction preparation, signing, finality, proof, and verification.
- Do not include private network topology, private RPC endpoints, secrets, or local paths.
