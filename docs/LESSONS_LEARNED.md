# Lessons Learned

## Wallet boundaries need to be visible

Users should be able to tell when the system is preparing a transaction and when they are being asked to sign. Blurring that boundary creates risk.

## Finality is part of the product experience

Transaction status is not only a backend state. It affects user trust, error handling, and how the interface explains progress.

## Verification connects Web3 and AI workflows

The proof / receipt direction became a bridge from blockchain infrastructure toward AI artifact verification.

## Experimental infrastructure should be described carefully

It is important to show architecture and technical direction without implying public chain adoption, production scale, or unverified usage.
